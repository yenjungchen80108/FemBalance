# FemBalance — Data Dictionary

This document describes the cleaned, unified schema (`combined_clean.csv` /
`training_subset.csv`) produced by merging two source datasets:

- **dataset1** — real wearable sensor data (mcPHASES, PhysioNet), no outcome labels
- **dataset2** — synthetically augmented dataset with pre-computed outcome labels
  (`synthetic_augmented_flag = 1` for 100% of rows)

`training_subset.csv` is the subset where `hasLabel = True`, i.e. dataset2 only.
This is the only data used to train the anovulation prediction model.

| Column                 | Type      | Source        | Description                                                                                                                                                                                                                                     | Notes                                                                                                                                                                                                   |
| ---------------------- | --------- | ------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `participantId`        | string    | both          | Unique participant identifier, prefixed `ds1_` or `ds2_` to avoid ID collisions across the two source datasets                                                                                                                                  | Not a model feature                                                                                                                                                                                     |
| `dayInStudy`           | int       | both          | Day index relative to the start of each participant's tracking period                                                                                                                                                                           | Not a model feature; used for time-series ordering in the UI                                                                                                                                            |
| `age`                  | int       | both          | Participant age in years                                                                                                                                                                                                                        | **Model input.** For dataset1, computed as `2025 − birth_year`                                                                                                                                          |
| `heartRate`            | float     | dataset1 only | Heart rate (bpm), from wearable device. `bpm` used first, falling back to `resting_heart_rate`, then `value` (from `resting_heart_rate.csv`) if missing                                                                                         | Not available in dataset2 — **not used in training**. Rows with `heartRate == 0` (123 rows, 2.2%) were treated as sensor dropout and set to missing                                                     |
| `skinTemp`             | float     | both          | Skin temperature signal, wearable device. **dataset1**: `temperature_diff_from_baseline` — deviation from personal baseline, °C (see `wrist_temperature.csv` in the mcPHASES README). **dataset2**: `BBT` — absolute basal body temperature, °C | **Model input.** ⚠️ Scale differs between sources — dataset1 values are centered near 0 (±2°C relative), dataset2 values are absolute (~36–37°C). Do not plot both sources on the same temperature axis |
| `lh`                   | float     | both          | Luteinizing hormone (mIU/mL), urine hormone test                                                                                                                                                                                                | **Model input.** Values > 50 mIU/mL (36 rows, 0.6% of dataset1) were treated as outliers and set to missing                                                                                             |
| `estrogen`             | float     | both          | Estrone-3-glucuronide (E3G), estrogen metabolite (ng/mL), urine hormone test                                                                                                                                                                    | **Model input**                                                                                                                                                                                         |
| `phase`                | string    | both          | Menstrual cycle phase label (Follicular / Ovulation-Fertility / Luteal / Menstrual)                                                                                                                                                             | Descriptive only, not used as a model input (would leak the target)                                                                                                                                     |
| `anovulationFlag`      | int (0/1) | dataset2 only | Whether this day is a suspected anovulatory cycle day                                                                                                                                                                                           | **Model target (label).** Sourced from `anovulation_proxy` in dataset2. Missing (`NA`) for dataset1 since no ground truth exists                                                                        |
| `cycleRegularityScore` | float     | dataset2 only | Cycle-to-cycle length variability, sourced from `cycle_variability`                                                                                                                                                                             | Statistically computed, not model-predicted. Missing for dataset1                                                                                                                                       |
| `interCycleLength`     | int       | dataset2 only | Length of the participant's menstrual cycle in days (constant per participant, not day-varying)                                                                                                                                                 | Missing for dataset1. Used to derive `cycleRegularityScore` context in the UI                                                                                                                           |
| `flagged`              | bool      | both          | Composite risk flag. **dataset2**: `True` if a participant's overall anovulation rate exceeds 40% across their record. **dataset1**: always `False` (no label to base a flag on)                                                                | Drives the risk badge on the Patient List page                                                                                                                                                          |
| `dataSource`           | string    | both          | `dataset1_real_wearable` or `dataset2_synthetic`                                                                                                                                                                                                | Used to branch UI behavior (e.g. show "No Model Prediction" for dataset1)                                                                                                                               |
| `isSynthetic`          | bool      | both          | Whether the row originates from synthetically augmented data                                                                                                                                                                                    | `False` for all dataset1 rows, `True` for all dataset2 rows                                                                                                                                             |
| `hasLabel`             | bool      | both          | Whether this row has a usable `anovulationFlag` for supervised training                                                                                                                                                                         | `True` only for dataset2. Filters `training_subset.csv`                                                                                                                                                 |

## Known limitations

- **Training data is 100% synthetic.** The only rows with a usable target label
  (`anovulationFlag`) come from dataset2, where `synthetic_augmented_flag = 1`
  for every row. Real-world wearable data (dataset1) has no ground truth and is
  used for display purposes only, not for training.
- **Small sample size.** Only 42 participants contribute to model training
  (2,937 daily records). Cross-validation is grouped by `participantId` to
  avoid data leakage, but the small number of unique individuals limits how
  confidently the results generalize.
- **No heart rate in training data.** Because dataset2 lacks `heartRate`, the
  trained model uses only `age`, `skinTemp`, `lh`, and `estrogen` as inputs —
  heart rate is displayed in the UI for context (dataset1 patients) but does
  not factor into the anovulation prediction.
- **Temperature scale mismatch across sources.** `skinTemp` means different
  things in each dataset (relative deviation vs. absolute BBT) and should not
  be visually compared across `dataset1` and `dataset2` patients on the same axis.
